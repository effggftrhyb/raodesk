# RAODESK Pilot

[`RAODESK`](RAODESK) — статический пилотный ITSM / Helpdesk сервис для управляющих компаний и жилищных служб.

Демо собрано так, чтобы его можно было показать через [`GitHub Pages`](RAODESK/README.md) без backend-части, но при этом покрыть основные процессы пилота service desk / ITSM.

## Что реализовано в пилоте

### 1. Фейковая авторизация

Вход через 3 демонстрационные учётные записи:

- `admin@raodesk.demo` / `Admin123!` — главный администратор
- `dispatcher@raodesk.demo` / `Dispatch123!` — диспетчер / оператор
- `specialist@raodesk.demo` / `Special123!` — специалист участка

Реализация находится в [`script.js`](RAODESK/script.js).

### 2. Приём и регистрация обращений

В пилоте показаны каналы:

- телефон
- email
- портал самообслуживания
- чат-бот / мессенджеры
- форма на сайте
- мониторинг

Есть форма создания новой заявки с автоматическим присвоением номера, типа обращения и SLA. Оператор может создать обращение от имени клиента.

### 3. Классификация и приоритизация

Поддержаны демонстрационные поля:

- категория
- услуга
- приоритет
- сроки SLA
- исполнитель
- команда

### 4. Управление жизненным циклом заявок

Поддержаны состояния:

- `Новая`
- `Назначена`
- `В работе`
- `На уточнении`
- `На контроле`
- `Закрыта`

Есть архивный признак и таймлайн изменений.

### 5. Коммуникации и уведомления

Реализованы:

- публичные комментарии в тикете
- внутренние комментарии для команды
- модальное окно уведомлений
- моделирование email / SMS / push / чат-бот уведомлений

### 6. Портал самообслуживания и база знаний

Показаны:

- история обращений клиента
- база знаний
- FAQ
- шаблоны ответов

### 7. SLA и эскалации

В интерфейсе есть:

- нормативы первого ответа
- нормативы решения
- риск нарушения SLA
- сценарии эскалации
- контроль опросов качества

### 8. Аналитика и KPI

В пилоте предусмотрены:

- объём заявок
- среднее время реакции
- среднее время решения
- выполнение в срок
- нагрузка специалистов
- повторные обращения
- оценка качества сервиса

### 9. Интеграции

Показаны интеграционные направления:

- корпоративная почта
- CRM / ERP / 1С
- мессенджеры и чат-боты
- мониторинг инженерных систем
- ITSM-расширения

### 10. Дополнительные возможности

В интерфейсе учтены:

- эскалация заявок
- учёт активов
- мобильный сценарий для выездных сотрудников
- двухфакторная аутентификация как блок готовности
- журнал аудита действий

## Локальный запуск

Это статический проект. Достаточно открыть [`index.html`](RAODESK/index.html) в браузере.

## Деплой на GitHub Pages

1. Создать репозиторий на GitHub.
2. Загрузить в него содержимое папки [`RAODESK`](RAODESK).
3. Проверить, что [`index.html`](RAODESK/index.html) находится в корне репозитория.
4. В GitHub открыть `Settings → Pages`.
5. Выбрать:
   - `Source` → `Deploy from a branch`
   - `Branch` → `main`
   - `Folder` → `/ (root)`
6. После публикации демо станет доступно как публичный сайт.

## Важное замечание

Текущая версия — это именно статический пилот / demo prototype. Для production-версии понадобятся:

- backend
- база данных
- реальная аутентификация
- серверные интеграции
- хранение файлов и вложений
- реальная отправка уведомлений
- управление правами доступа на сервере
